# Vibesドキュメント規約

## ディレクトリ構成

| ディレクトリ | 役割 | 変更頻度 |
|-------------|------|---------|
| **rules/** | プロジェクト規約 (コーディング規約、アーキテクチャ) | チーム合意時のみ |
| **specs/** | 技術仕様 (システム仕様、アーキテクチャ設計) | アーキテクチャ変更時のみ |
| **tasks/** | 作業手順書 (手順書、チェックリスト、ガイド) | 実作業に応じて柔軟に更新 |
| **temps/** | 一時ドキュメント (進行中タスク、検討事項) | 作業完了後削除 |

**Note**: `apis/` (外部API仕様)、`logics/` (機能設計書) は廃止されました。
API仕様はTypeScript型定義 (`types/normalized-api.ts`) で管理します (Code as Document)。

## 基本原則

- **簡潔性**: 専門用語活用、情報集約、圧縮言語での記載
- **Code as Document**: 型定義・テストコードが仕様、マークダウンは最小限
- **責務分離**: バージョン管理はGit、ドキュメントは内容に集中
